# Schyotovod

Сервис автоматической выгрузки счетов из почты Gmail в задачу Pyrus.

Ежемесячно в заданный период (по умолчанию с 20 по 29 число) сервис отслеживает почтовый ящик Gmail на предмет писем со счетами от определённого отправителя, скачивает вложения и прикрепляет их в поле типа «Вложение» конкретной задачи Pyrus.

## Установка и запуск на Ubuntu (в качестве systemd-службы)

Для быстрой установки выполните команду:
```bash
curl -sSL https://raw.githubusercontent.com/ho154/schyotovod/main/deploy/install.sh | sudo bash
```

Установка в произвольную директорию (по умолчанию используется `/opt/schyotovod`):
```bash
curl -sSL https://raw.githubusercontent.com/ho154/schyotovod/main/deploy/install.sh | sudo bash -s -- /your/custom/path
```

Скрипт автоматически:
1. Создаст беспривилегированного системного пользователя `schyotovod`.
2. Скачает последний релиз бинарного файла и проверит его контрольную сумму SHA256.
3. Интерактивно запросит создание логина и пароля администратора для доступа к веб-панели управления.
4. Настроит, зарегистрирует и запустит systemd-службу `schyotovod.service`.

После этого панель настроек и диагностики будет доступна по адресу `http://<ip-адрес-сервера>:47291`.

### Управление службой в Ubuntu:
```bash
systemctl status schyotovod       # статус службы
journalctl -u schyotovod -f       # логи реального времени
schyotovod --reset-admin-password # интерактивный сброс логина и пароля
schyotovod --version              # посмотреть версию программы
```

## Установка и запуск на Windows / Windows Server

1. **Скачайте исполняемый файл**:
   Перейдите в раздел [Releases](https://github.com/ho154/schyotovod/releases) и скачайте актуальный бинарный файл `schyotovod_windows_amd64.exe` (или `arm64`), а также файл контрольной суммы.
2. **Инициализация администратора**:
   Откройте PowerShell от имени администратора в папке с программой и запустите команду для создания первого администратора панели:
   ```powershell
   .\schyotovod_windows_amd64.exe --data-dir .\data --init-admin-login admin --init-admin-password admin
   ```
3. **Запуск сервиса в консоли**:
   Вы можете запустить бота напрямую в терминале:
   ```powershell
   .\schyotovod_windows_amd64.exe --data-dir .\data
   ```
4. **Установка в качестве системной службы (Windows Server)**:
   Для запуска в фоновом режиме на Windows Server рекомендуется использовать [NSSM (Non-Sucking Service Manager)](https://nssm.cc/):
   - Скачайте nssm.
   - Запустите команду: `nssm install Schyotovod`.
   - В появившемся окне укажите путь к `schyotovod_windows_amd64.exe` и аргумент `--data-dir C:\Schyotovod\data`.
   - Нажмите "Install service" и запустите его в службах Windows (`services.msc`).

## Основной функционал

- **Мгновенная реакция на новое письмо** через **IMAP IDLE** + резервный поллинг (актуально для Gmail).
- **Интеграция с Pyrus API v4**: поиск задач клиентов по форме Pyrus в расчетный период, прикрепление файлов вложений в поле типа «Вложение» и обновление суммы в поле «Сумма».
- **Динамический фильтр по периоду**: поиск задач по форме оптимизирован и фильтруется на стороне Pyrus по дате создания (`created_after` на основе расчетного периода), что минимизирует объем передаваемых данных.
- **Интеллектуальная замена вложений**: при получении нового счета, если в задаче Pyrus в поле вложений уже прикреплены файлы, бот прикладывает новый файл и оставляет комментарий с указанием общего количества ранее загруженных файлов во избежание путаницы, вместо очистки поля.
- **Отказоустойчивость и ретраи**:
  - Логика попыток и таймеры повторных обращений к API сохраняются при перезапусках бота благодаря хранению состояния попыток (`attempts.json`) на диске.
  - Состояние попыток привязывается к уникальному `Message-ID` письма, поэтому приход нового письма сбрасывает счетчики ретраев для клиента, а повторное чтение того же ошибочного письма не обнуляет таймер ожидания ретрая.
- **Веб-панель** на русском: настройки с подсказками к каждому полю, просмотр логов с фильтрацией по уровням, кнопка «Проверить сейчас» (ручной запуск проверки почты), «Проверить доступ к почте» и «Проверить доступ к Pyrus».
- **Проверка настроек при сохранении** реальными запросами к Gmail и Pyrus — ошибки показываются под конкретными полями на русском.
- **Дедупликация писем**: обработанные письма регистрируются по `Message-ID` в JSON-файле и не обрабатываются повторно. Файл автоматически очищается при переходе на новый календарный месяц.
- **Удобная диагностика**: кнопка «Проверить поля в форме Pyrus» позволяет протестировать соответствие полей и выводит полный JSON-ответ шаблона формы Pyrus без ограничений по размеру строк в лог-файл (при клике временно включается режим Debug на максимум).
- **Самообновление** с GitHub Releases: проверка контрольной суммы SHA256, self-check новой версии и автоматический откат при неудаче.

## Настройки конфигурации в Панели

### Почта (Gmail)
- **Email** (`gmail.email`) — адрес почты Gmail.
- **Пароль приложения** (`gmail.app_password`) — пароль приложения Gmail (16-символьный пароль, пробелы при вводе автоматически вырезаются).
- **IMAP сервер** (`gmail.imap_host`) — адрес IMAP-сервера. По умолчанию `imap.gmail.com`.
- **IMAP порт** (`gmail.imap_port`) — порт IMAP-сервера. По умолчанию `993`.

### Отбор писем и период проверки
- **Отправитель счета** (`filter.sender_email`) — e-mail, с которого приходят счета.
- **День начала периода** (`filter.start_day`) — день месяца, с которого начинается отслеживание (например, `20`).
- **День окончания периода** (`filter.end_day`) — день месяца, по который ведется отслеживание (например, `29`).
- **Интервал поллинга** (`filter.fallback_poll_interval_minutes`) — интервал резервной проверки почты (если IMAP IDLE разорван).

### Pyrus
- **Логин бота** (`pyrus.login`) — e-mail бота Pyrus.
- **Ключ безопасности** (`pyrus.security_key`) — секретный ключ бота из настроек Pyrus.
- **ID формы Pyrus** (`pyrus.form_id`) — числовой идентификатор формы.
- **ID поля-вложения** (`pyrus.attachment_field_id`) — Field ID поля типа «Вложение» в форме.
- **ID поля наименования клиента** (`pyrus.client_name_field_id`) — Field ID текстового поля с названием клиента.
- **ID поля суммы** (`pyrus.amount_field_id`) — Field ID поля типа «Сумма» (деньги или число).
- **Максимум попыток обновления** (`pyrus.max_update_attempts`) — лимит попыток обновления при ошибках API.
- **Интервал повтора при ошибке** (`pyrus.retry_interval_minutes`) — время ожидания перед повторной попыткой.

### Общие
- **Часовой пояс** (`general.timezone`) — таймзона для расчетов периодов и записи времени логов (например, `Asia/Almaty`).
- **Срок хранения логов в днях** (`general.log_retention_days`) — время жизни файлов логов (logs/YYYY-MM-DD.log).

## Сброс доступа к панели

Если вы забыли пароль администратора, выполните на сервере:
- **Ubuntu**: `sudo schyotovod --reset-admin-password`
- **Windows**: `.\schyotovod_windows_amd64.exe --data-dir .\data --reset-admin-password`

Команда сгенерирует временный пароль и выведет его в консоль. При первом входе с этим паролем система потребует задать новый постоянный пароль.

## Безопасность

- Все секреты сохраняются в файле `config.json` с правами доступа `600`.
- В логах безопасности и отладки все секретные ключи и пароли приложений автоматически маскируются.
